# Docs Index

This folder contains supporting project workflow and handoff docs.

## Available Docs

- `architecture-overview.md`: High-level app structure, flow, services, and database shape
- `database-overview.md`: Supabase schema structure, core tables, and notification pipeline
- `feature-overview.md`: Feature-by-feature map of screens, services, and data sources
- `service-overview.md`: Shared services and local helper responsibilities
- `release-process.md`: High-level flow for preparing and shipping a release
- `release-checklist.md`: Pre-release and schema rollout verification steps
- `handoff-template.md`: Reusable summary template for feature or release handoff
- `testing-notes.md`: Manual QA session notes template
- `ci-checklist.md`: CI planning and automation checklist
- `troubleshooting.md`: Common local setup and workflow fixes

## Suggested Order

1. Use `architecture-overview.md` when you need a quick map of the app.
2. Use `database-overview.md` when you need a quick map of the schema and trigger pipeline.
3. Use `feature-overview.md` when you need a module-by-module map.
4. Use `service-overview.md` when you need a quick map of shared responsibilities.
5. Use `release-process.md` when preparing to ship.
6. Use `release-checklist.md` when validating a release.
7. Use `testing-notes.md` while running manual QA.
8. Use `handoff-template.md` when handing work to the next person.
9. Use `ci-checklist.md` when expanding or maintaining automation.
10. Use `troubleshooting.md` when a local setup or workflow command fails.
