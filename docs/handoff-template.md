# Handoff Template

Use this template when handing off a feature, fix, or release-ready change.

## Summary

- What changed?
- Why was it changed?

## Files Or Areas Touched

- `path/to/file`
- `another/path`

## Verification

- What was tested?
- What commands were run?
- What was verified manually in the app?

## Risks Or Follow-Ups

- Known gaps
- Areas that still need validation
- Reasonable next steps

## Notes For The Next Person

- Required environment variables
- Schema or migration considerations
- Setup details, caveats, or temporary workarounds
